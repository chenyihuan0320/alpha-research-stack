# 开源项目选型审计模板

本文件是 Alpha Research Stack 的开源项目选型审计模板。当前阶段只建立模板，不做联网审计，不记录未经验证的最新状态。

## 审计原则

- 优先评估能直接提升数据可靠性、研究效率、回测严谨性和信号复盘质量的项目。
- 不因为项目知名就默认引入，必须评估维护状态、许可证、集成成本和替代方案。
- 不把示例 notebook 或 demo 当作生产能力。
- 对涉及模型、LLM 或智能体的项目，必须区分“研究辅助能力”和“最终 confidence 生成能力”。最终 confidence 不能由 LLM 主观给出。

## 通用审计维度

每个项目按以下维度记录：

- 项目名称：
- 仓库或官网：
- 许可证：
- 主要用途：
- 对本项目的潜在角色：
- 支持市场：A 股 / 港股 / 美股 / 跨市场 / 其他
- 数据覆盖：
- 核心能力：
- 成熟度与维护状态：
- 文档质量：
- 安装和运行复杂度：
- 与 Python / Node / 数据库 / 工作流的集成方式：
- 可复用模块：
- 需要二次开发的部分：
- 已知限制：
- 安全与合规风险：
- 替代方案：
- 初步结论：采用 / 观察 / 不采用 / 待验证
- 下一步验证任务：

## 审计输出格式

正式审计完成后，每个项目应输出：

- 一句话结论。
- 推荐使用场景。
- 不推荐使用场景。
- 集成优先级。
- 最小验证任务。
- 替代方案。
- 是否进入 Alpha Research Stack 初始技术栈。
